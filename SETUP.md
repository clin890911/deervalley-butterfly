## 完整測試（免設定，Demo 模式）

直接開上線網址或本機 `index.html` 就能整套跑一遍：
1. 填**姓名／電話／選一張照片** → 送出，看到「登記完成」與 LINE 按鈕。
2. 按「**＋ 再登記一位**」可連續登記多筆（現場一機多客也用得到）。
3. 頁尾「**查看已收到的登記（Demo 後台）**」→ 收件匣列出每筆的**照片縮圖＋姓名＋電話＋時間**，模擬正式版試算表會看到的內容。
4. 後台可「**匯出 CSV**」或「**全部清空**」重測。
> Demo 模式：資料只暫存在該裝置瀏覽器（localStorage），**不會上傳**。接上後端後，正式版會自動隱藏這個 Demo 後台入口。

## 接後端（零設定版，約 5 分鐘）

腳本綁在報名試算表上，**不需要複製任何 ID**。

### 步驟 1 — 建立報名試算表
開一份新的 Google 試算表，命名例如「蝶影報名」。標題列不用自己打，後端第一次寫入時會自動補上（時間／姓名／電話／照片連結）。

### 步驟 2 — 從試算表開 Apps Script
在這份試算表上方選單：**擴充功能 → Apps Script**。會開啟一個綁定在此試算表的腳本專案。

### 步驟 3 — 貼上後端程式
把 `apps-script/Code.gs` 內容**整段**貼進去（覆蓋預設的 `myFunction`），存檔（Ctrl/Cmd + S）。不用改任何一行。

### 步驟 4 — 部署成網頁應用程式
1. 右上「**部署 → 新增部署作業**」。
2. 齒輪選「**網頁應用程式**」。
3. 設定：執行身分「**我（你的帳號）**」、誰可以存取「**所有人**」。
4. 「部署」→ 第一次要求授權，一路允許（若出現「Google 尚未驗證這個應用程式」→ **進階 → 前往專案（不安全）→ 允許**；這是你自己的腳本，安全）。
5. 複製畫面上的 **網頁應用程式網址**（結尾是 `/exec`）。

### 步驟 5 — 把 /exec 網址給我（或自己填）
把網址貼進 `index.html` 最上方：
```js
const GOOGLE_SCRIPT_URL = "步驟4複製的 /exec 網址";
```
存檔後重新 push 到 GitHub，上線網址就會自動改成「真的收件」。（LINE 連結已填好，不用動。）
